## 计划：提交分支代码并拉取主分支代码

### 1. 检查当前git状态

* 执行 `git status` 命令，查看当前分支状态和未提交的更改

* 确认所有更改都已准备好提交

### 2. 提交当前更改

* 执行 `git add .` 命令，将所有更改添加到暂存区

* 执行 `git commit -m "完成WxBotGo账号封禁修复、ID持久化、二维码扫描体验改进、WeComBot重命名为WeWorkBot以及中文文档支持"` 命令，提交更改到本地分支

### 3. 拉取主分支最新代码

* 执行 `git pull origin main` 命令，尝试拉取origin/main的最新代码

### 4. 处理冲突（如果有）

* 如果拉取过程中出现冲突，查看冲突文件

* 手动解决冲突，确保代码的完整性和正确性

* 提交冲突解决后的更改

### 5. 验证最终状态

* 执行 `git status` 命令，确认代码已成功更新

* 执行 `git log --oneline -n 5` 命令，查看最近的提交记录

### 注意事项

* 确保在提交前检查所有更改，避免提交不必要的文件

* 如果拉取失败，可能需要检查网络连接或GitHub仓库状态

* 冲突解决时，确保保留所有必要的更改
