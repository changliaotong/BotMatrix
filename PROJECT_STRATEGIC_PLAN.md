## 1. 核心商业哲学：谁在为价值买单？

我们将 BotMatrix 的价值拆解为三个维度，针对不同客群实施差异化收割策略：

| 维度 | 核心价值 | 收费对象 | 定价逻辑 |
| :--- | :--- | :--- | :--- |
| **L1: 算力与基座层** | 连接、带宽、基础调度 | 开发者、中小型 Bot 运营者 | 资源消耗型 (按量付费) |
| **L2: 专家智能体层** | 垂直领域的业务结果 (如：订单、线索) | 跨境卖家、量化团队、自媒体 | 价值增益型 (订阅 + 佣金) |
| **L3: 企业神经中枢** | 流程自动化 (RPA)、私有化安全 | 大中型企业、政府机构 | 解决方案型 (定制费 + 维保) |

## 3. 极致落地的技术支撑 (The "Guts")

为了让收钱变得自动化、透明化，我们需要在代码层面落实：

### 3.1 Matrix-Wallet (统一结算系统)
*   在 [BotNexus](file:///d:/projects/BotMatrix/src/BotNexus) 中集成虚拟钱包。
*   **计费钩子**: 在 `dispatcher.go` 中加入计费拦截器。每一条通过 Neural Nexus 路由的消息，根据其背后的 Agent 价值，实时扣除用户额度。

### 3.2 证据链存证 (Evidence Chain)
*   利用 [PostgreSQL](file:///d:/projects/BotMatrix/migration_objects.sql) 记录每一笔 Agent 的“业务贡献”。例如：Agent 成功引导了一个支付链接，系统自动打标并触发佣金结算。

### 3.3 插件系统 2.0 路线图 (伟大的未来)

#### 第一阶段：基础设施夯实 (Current)
- [x] 多语言 SDK (Go, Python, C#) 对齐。
- [x] 声明式权限系统 (plugin.json)。
- [x] 无状态分布式追踪 (Correlation ID)。
- [x] 分布式状态存储 (SessionStore)。
- [x] **bm-cli 开发工具链**：支持一键生成多语言插件模板。
- [x] **基础意图路由 (Intent Routing)**：支持基于关键字/正则的轻量级意图分发。
- [x] **BMPK 打包与热安装**：实现一键打包与 Core 层零重启插件安装。

#### 第二阶段：生态与分发 (Next)
- [x] **跨插件能力调用 (Skill Call)**：允许插件间通过 Core 进行 IPC 通信。
- [ ] **UI 插件系统 (Frontend Extensions)**：初步定义 UI 注入协议。
- [ ] **内置插件市场**：在 WebUI 中直接浏览与安装。
- [ ] **自动化审核流水线**：检测插件是否存在恶意 Action 调用。
- [ ] **多平台支持**：支持 Docker 化部署插件。

#### 第三阶段：能力边界突破 (2.0)
- [ ] **UI 插件 (Frontend Extensions)**：允许插件向 WebUI 注入自定义组件（如：管理面板）。
- [ ] **远程插件 (Remote Hooks)**：支持插件运行在远程服务器，通过 gRPC/WebSocket 与 BotNexus 通信。
- [ ] **跨插件依赖 (Plugin Interop)**：允许插件 A 依赖插件 B 导出的 Skill。
- [ ] **AI 增强插件 (AI-Native Plugins)**：原生集成 LLM，支持插件通过自然语言描述其功能并被 AI 自动编排。
